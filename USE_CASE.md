# Cas d’usage réel

## Titre
Assistant interne de support opérationnel pour équipe Produit/Tech.

## Problème
Les équipes perdent du temps à alterner entre:
- questions de compréhension technique,
- exécution de petites actions outillées,
- vérification de cohérence des réponses.

## Scénario
1. Un PM pose une question: "Quel est l’impact de ce changement API ?"
2. Le système classe l’intention (`query` ou `hybrid`).
3. Il combine récupération de contexte + raisonnement + garde-fous.
4. Il renvoie une réponse traçable avec métadonnées de phase/source.

## Résultat attendu
- Réponse plus rapide et homogène.
- Moins d’aller-retours entre équipes.
- Historisation potentielle des interactions utiles pour capitalisation.
